> **你的敌人不是市场，是你自己的情绪。**

## 二、盘前操作（08:30-09:25）

### Step 1：运行系统（1分钟）
```bash
cd d:\workspace\trading-system\trading_system
python main.py
```
等待运行完成（约80秒），系统会自动发送邮件。

### Step 2：查看邮件（3分钟）
你会收到 **5封邮件**，按优先级查看：

| 优先级 | 邮件标题 | 看什么 |
|--------|----------|--------|
| ★★★ | [东方财富] 条件单 | **今天必须挂的条件单**（止损/止盈/减仓） |
| ★★★ | [交易系统] 每日报告 | 信号明细（买入/卖出/观望） |
| ★★☆ | [仓位分析] | 风险预警（仓位/集中度） |
| ★☆☆ | [CANSLIM选股] | 观察池（当前满仓不用管） |
| ★☆☆ | [组合风控] | 整体风险评分 |

### Step 3：挂条件单（10分钟）
打开 **东方财富APP → 交易 → 条件单**，按邮件中的条件单逐一设置：

**设置方法：**
1. 定价卖出条件单：触发价 = 邮件中的止损价，委托价 = 触发价 × 0.99
2. 定价买入条件单：触发价 = 邮件中的买入价，委托价 = 触发价 × 1.01
3. 有效期选择"当日有效"

## 三、盘中铁律（09:30-15:00）

### ⛔ 绝对禁止：
- ❌ **禁止手动买入任何股票**（系统已锁定：仓位>90%无法买入）
- ❌ **禁止手动加仓浮亏股票**（铁则：浮亏不加仓）
- ❌ **禁止盘中临时起意操作**
- ❌ **禁止因为"感觉要涨"就下单**
- ❌ **禁止因为"跌了害怕"就割肉**（只看收盘价）

### ✅ 唯一允许：
- ✅ 条件单自动触发后确认成交
- ✅ 系统发出"强制卖出"信号后执行（单日跌>8%+放量）

### 🧠 情绪管理：
- 想操作时 → 关掉APP，去倒杯水
- 焦虑时 → 告诉自己"条件单会帮我处理"
- 想加仓时 → 打开持仓看看可用资金（¥142），你没钱了

## 四、盘后操作（15:30-16:00）

### Step 1：运行盘后分析
```bash
python main.py --no-update
```
（如果盘前已运行且数据已更新，可加 `--no-update` 跳过数据拉取）

### Step 2：检查邮件
- 看选股结果（观察，不操作）
- 看持仓诊断（是否需要明天减仓）
- 看明日条件单（重复Step 3）

### Step 3：更新持仓（如有变动）
如果今天有成交，编辑 `d:\workspace\trading-system\holdings.json`：
```json
{
  "股票代码": {
    "shares": 股数,
    "buy_price": 成本价,
    "highest_price": 持仓期间最高价,
    "first_batch_done": true,
    "sector": "赛道名",
    "stock_type": "龙头/弹性"
  }
}
```

## 六、买入后持仓管理（中线波段 1~6 周）

> **系统定位：中线波段交易，持仓周期通常 1~6 周。**
> **核心原则：买入后你不需要自己判断卖点，系统每天自动帮你决定。**

### 6.1 系统风格参数

| 维度 | 参数 | 说明 |
|------|------|------|
| 入场依据 | MA20回踩 + 放量突破确认 | 中线入场信号 |
| 趋势参考 | MA20(短期) + MA60(中期) | 2~12周波段 |
| 初始止损 | -10% | 买入价下方10% |
| 止盈目标 | +8%卖1/3, +20%卖1/3 | 阶梯止盈 |
| 移动止损 | 浮盈>30%锁22%利润 | 让利润奔跑 |

### 6.2 买入成交后你要做什么？

```
Day 0: 条件单触发买入成交
       ↓ 立即做两件事：
       ① 更新 holdings.json（加入新持仓）
       ② 挂初始止损条件单（买入价 × 0.90）
       ↓
Day 1起: 每天盘后运行 python main.py
       ↓
系统自动判断:
  ├── 没触发卖出 → 邮件显示"[持仓观望] 浮盈X%, 移动止损=XX.XX"
  │                → 你什么都不用做，继续持有
  │
  └── 触发卖出 → 邮件显示"[卖出] 原因..."
                  → 第二天开盘挂卖出条件单（或市价卖）
```

### 6.3 系统自动触发的5个卖出条件（按优先级）

| 优先级 | 卖出条件 | 说明 | 紧急度 |
|--------|---------|------|--------|
| 1 | 单日放量大跌>8% | 资金出逃，无条件离场 | 次日开盘市价卖 |
| 2 | 收盘跌破移动止损线 | 见下方阶梯表 | 次日挂条件单 |
| 3 | 跌破60日线 + 均线拐头 | 趋势结束 | 次日挂条件单 |
| 4 | MACD死叉（盈利状态） | 上涨动能衰竭 | 观察1天确认 |
| 5 | 从最高点回落超阈值 | 龙头回落5%/弹性回落3% | 次日挂条件单 |

### 6.4 移动止损阶梯（自动上移，只升不降）

| 浮盈区间 | 止损位 | 含义 | 举例(买入价¥100) |
|---------|--------|------|------------------|
| 0~5% | 买入价 × 0.90 | 初始保护（亏10%走人） | 止损¥90 |
| 5%~15% | 买入价 × 1.02 | 保本+2%（不白忙） | 止损¥102 |
| 15%~30% | 买入价 × 1.12 | 锁定12%利润 | 止损¥112 |
| >30% | 买入价 × 1.22 | 锁定22%利润 | 止损¥122 |

> 关键：止损价只会往上移，永远不会下移。一旦上移就锁定了利润。

### 6.5 阶梯止盈（分批卖出）

| 浮盈阈值 | 操作 | 说明 |
|---------|------|------|
| +8% | 卖出 1/3 仓位 | 先落袋一部分 |
| +20% | 再卖出 1/3 仓位 | 锁定大部分利润 |
| 剩余 1/3 | 移动止损跟踪 | 让利润奔跑，直到回落止盈触发 |

### 6.6 完整持仓生命周期示例

假设条件单买入迈瑞医疗 ¥151.83（200股）：

| 天数 | 收盘价 | 浮盈 | 系统判断 | 你的操作 |
|------|--------|------|---------|----------|
| Day 0 | 151.83 | 0% | 建仓成功 | 挂止损单¥136.65 |
| Day 3 | 155.00 | +2% | 持仓观望 | **不动** |
| Day 7 | 160.00 | +5.4% | 止损上移至¥154.87 | 更新条件单止损价 |
| Day 12 | 170.00 | +12% | 持仓观望 | **不动** |
| Day 15 | 182.00 | +20% | 触发第一档止盈 | 卖1/3（约70股） |
| Day 20 | 175.00 | 回落3.8% | 未触发（龙头5%） | **不动** |
| Day 22 | 168.00 | 回落7.7% | 触发回落止盈 | 清仓剩余 |

> 总持仓约22天（3周），这就是典型的中线波段节奏。

### 6.7 持仓期间铁律

- ❌ **浮亏时绝对不加仓**（系统已锁定，浮亏加仓会被风控拒绝）
- ❌ **不因为"跌了害怕"提前止损**（只看收盘价，不看盘中波动）
- ❌ **不因为"涨了一点"就急着卖**（系统没说卖就不卖）
- ❌ **不手动修改止损价往下移**（止损只能上移）
- ✅ **每天运行系统，看邮件，按信号操作**
- ✅ **止损价上移时，更新条件单**
- ✅ **系统说卖就卖，不犹豫**

### 6.8 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 买入后连续跌停 | 第一个跌停板挂卖出条件单，不抱幻想 |
| 利好消息大涨 | 不追加，等系统止盈信号 |
| 利空消息大跌 | 看是否触发强制卖出（-8%+放量），否则等收盘 |
| 横盘超过3周 | 系统会检测趋势破位，自动给出卖出信号 |
| 大盘暴跌 | 系统有日度/周度熔断，会自动收紧止损 |
